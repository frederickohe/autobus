# Blotato Social Media Integration Examples

## Setup Instructions

### 1. Environment Variables

Add these to your `.env` file:

```bash
# Blotato Configuration
BLOTATO_API_KEY=your_blotato_api_key
BLOTATO_CLIENT_ID=your_blotato_client_id
BLOTATO_CLIENT_SECRET=your_blotato_client_secret
BLOTATO_API_BASE=https://api.blotato.com
BLOTATO_OAUTH_BASE=https://app.blotato.com
BASE_FRONTEND_URL=http://localhost:3000
```

### 2. Database Migration

Run Alembic migrations to create the `social_accounts` table:

```bash
cd /path/to/autobus
alembic upgrade head
```

## API Endpoints

### 1. Initiate Account Connection (OAuth)

**Endpoint:** `GET /api/v1/social/connect/{platform}`

**Description:** Start OAuth flow for connecting a social media account

**Parameters:**
- `platform` (path): Social media platform name (TWITTER, LINKEDIN, FACEBOOK, INSTAGRAM, TIKTOK, YOUTUBE, THREADS, BLUESKY, MASTODON)

**Headers:**
- `Authorization: Bearer {JWT_TOKEN}` - Authenticated user JWT token

**Example Request:**
```bash
curl -X GET "http://localhost:8000/api/v1/social/connect/TWITTER" \
  -H "Authorization: Bearer your_jwt_token"
```

**Response:**
```json
{
  "authorization_url": "https://app.blotato.com/oauth/authorize?response_type=code&client_id=...",
  "platform": "TWITTER",
  "message": "Redirect user to this URL to authorize account connection"
}
```

**Flow:**
1. User clicks the returned `authorization_url`
2. User authorizes on Blotato
3. User is redirected to `/api/v1/social/callback` with `code` and `state`

---

### 2. OAuth Callback Handler

**Endpoint:** `GET /api/v1/social/callback`

**Description:** Handles OAuth callback from Blotato after user authorization

**Query Parameters:**
- `code` (string): Authorization code from Blotato
- `state` (string): State for CSRF validation
- `error` (optional string): Error code if authorization failed
- `error_description` (optional string): Error description

**Note:** This is handled automatically by Blotato redirecting the user

**Example Callback URL (generated by Blotato):**
```
http://localhost:3000/api/v1/social/callback?code=auth_code_xyz&state=state_xyz
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Account connected successfully",
  "platform": "TWITTER",
  "account_name": "@john_doe",
  "redirect_url": "http://localhost:3000/social/accounts?connected=true"
}
```

**Response (Error):**
```json
{
  "success": false,
  "error": "Invalid or expired state parameter"
}
```

### 4. Disconnect Account

**Endpoint:** `DELETE /api/v1/social/accounts/{account_id}`

**Description:** Disconnect a social media account (local database only)

**Parameters:**
- `account_id` (path): ID of the account to disconnect

**Headers:**
- `Authorization: Bearer {JWT_TOKEN}`

**Request Body:**
```json
{
  "disconnect_from_blotato": false
}
```

**Example Request:**
```bash
curl -X DELETE "http://localhost:8000/api/v1/social/accounts/sa_123456" \
  -H "Authorization: Bearer your_jwt_token" \
  -H "Content-Type: application/json" \
  -d '{"disconnect_from_blotato": false}'
```

**Response:**
```json
{
  "success": true,
  "message": "Account disconnected successfully",
  "account_id": "sa_123456"
}
```

## Python Integration Examples

### Example 1: Basic Post Publishing

```python
import httpx
import json

async def post_to_twitter(user_jwt_token: str, content: str):
    """Post content to user's Twitter account"""
    
    async with httpx.AsyncClient() as client:
        # Get user's accounts
        accounts_response = await client.get(
            "http://localhost:8000/api/v1/social/accounts",
            headers={"Authorization": f"Bearer {user_jwt_token}"}
        )
        
        accounts_data = accounts_response.json()
        twitter_accounts = [
            acc for acc in accounts_data["accounts"] 
            if acc["platform"] == "TWITTER"
        ]
        
        if not twitter_accounts:
            return {"error": "No Twitter account connected"}
        
        account_id = twitter_accounts[0]["id"]
        
        # Publish post
        publish_response = await client.post(
            "http://localhost:8000/api/v1/social/post",
            headers={"Authorization": f"Bearer {user_jwt_token}"},
            json={
                "account_ids": [account_id],
                "content": content,
                "media_urls": None,
                "schedule_time": None,
                "hashtags": None
            }
        )
        
        return publish_response.json()


# Usage
result = await post_to_twitter("user_jwt_token", "Hello Twitter!")
if result.get("success"):
    print(f"Posted successfully! Post ID: {result['results'][0]['post_id']}")
else:
    print(f"Failed: {result['message']}")
```

### Example 3: Scheduled Post Publishing

```python
from datetime import datetime, timedelta

async def schedule_post(user_jwt_token: str, content: str, account_ids: list):
    """Schedule a post for future publication"""
    
    # Schedule 24 hours from now
    schedule_time = (datetime.utcnow() + timedelta(hours=24)).isoformat() + 'Z'
    
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8000/api/v1/social/post",
            headers={"Authorization": f"Bearer {user_jwt_token}"},
            json={
                "account_ids": account_ids,
                "content": content,
                "media_urls": None,
                "schedule_time": schedule_time,
                "hashtags": None
            }
        )
        
        return response.json()


# Usage
result = await schedule_post(
    jwtToken, 
    "This post will be published tomorrow!",
    ["sa_123456"]
)
```

### Example 4: Multi-Platform Campaign

```python
async def launch_campaign(user_jwt_token: str, campaign_content: str):
    """Launch a campaign across all connected platforms"""
    
    async with httpx.AsyncClient() as client:
        # Get all connected accounts
        accounts_response = await client.get(
            "http://localhost:8000/api/v1/social/accounts",
            headers={"Authorization": f"Bearer {user_jwt_token}"}
        )
        
        accounts = accounts_response.json()["accounts"]
        account_ids = [acc["id"] for acc in accounts if acc["is_active"]]
        
        if not account_ids:
            return {"error": "No connected accounts"}
        
        # Publish to all platforms
        campaign_response = await client.post(
            "http://localhost:8000/api/v1/social/post",
            headers={"Authorization": f"Bearer {user_jwt_token}"},
            json={
                "account_ids": account_ids,
                "content": campaign_content,
                "media_urls": None,
                "schedule_time": None,
                "hashtags": ["campaign", "launch"]
            }
        )
        
        result = campaign_response.json()
        
        # Log the campaign results
        print(f"Campaign launched: {result['successful_posts']}/{result['total_platforms']} successful")
        
        return result
```

## Chatbot Integration Example

```python
from core.socialmedia.service.socialmedia_service import SocialMediaService
from core.socialmedia.service.post_publishing_service import PostPublishingService
from core.socialmedia.service.blotato_api_service import BlotatoAPIClient

# Inside your chatbot command handler
async def handle_post_command(user_id: str, command: str, args: list, db: Session):
    """
    Chatbot command: @bot post [platform] "content"
    Example: @bot post twitter "Check out our new feature!"
    """
    
    if len(args) < 2:
        return "Usage: @bot post [platform] \"content\""
    
    platform = args[0].upper()
    content = " ".join(args[1:]).strip('"')
    
    # Initialize services
    blotato_client = BlotatoAPIClient(
        api_key=BLOTATO_API_KEY,
        client_id=BLOTATO_CLIENT_ID,
        client_secret=BLOTATO_CLIENT_SECRET
    )
    
    social_service = SocialMediaService(db, blotato_client)
    
    # Get user's accounts for the platform
    accounts = social_service.get_user_accounts_by_platform(user_id, platform)
    
    if not accounts:
        return f"❌ Please connect your {platform} account first: /api/v1/social/connect/{platform}"
    
    # Publish post
    publishing_service = PostPublishingService(db, blotato_client)
    account_ids = [acc.id for acc in accounts]
    
    from core.socialmedia.dto.socialmedia_dto import PublishPostRequest
    publish_request = PublishPostRequest(
        account_ids=account_ids,
        content=content,
        media_urls=None,
        schedule_time=None
    )
    
    result = await publishing_service.publish_post(user_id, publish_request)
    
    if result.success:
        return f"✅ Posted successfully to {result.successful_posts} platforms! Post ID: {result.results[0].post_id if result.results else 'N/A'}"
    else:
        return f"❌ Failed to post: {result.message}"


# Another chatbot command: list accounts
async def handle_list_accounts_command(user_id: str, db: Session):
    """
    Chatbot command: @bot accounts
    Lists all connected social media accounts
    """
    
    blotato_client = BlotatoAPIClient(
        api_key=BLOTATO_API_KEY,
        client_id=BLOTATO_CLIENT_ID,
        client_secret=BLOTATO_CLIENT_SECRET
    )
    
    social_service = SocialMediaService(db, blotato_client)
    accounts = social_service.get_user_accounts(user_id)
    
    if not accounts:
        return "No connected accounts. Connect one: /api/v1/social/connect/{platform}"
    
    # Build response
    response = "📱 **Connected Accounts:**\n"
    for acc in accounts:
        status = "✅" if acc.is_active else "❌"
        response += f"\n{status} {acc.platform} - {acc.account_name}"
        if acc.last_used_at:
            response += f" (last used: {acc.last_used_at.strftime('%Y-%m-%d %H:%M')})"
    
    return response
```

## Error Handling

### Common Errors

#### 1. No Connected Accounts
```json
{
  "success": false,
  "error": "No connected accounts found",
  "detail": "Please connect an account first at /api/v1/social/connect/{platform}"
}
```

#### 2. Account Not Owned by User
```json
{
  "success": false,
  "error": "Account validation failed",
  "detail": "Account sa_xyz123 not found or not owned by user"
}
```

#### 3. Rate Limit Exceeded
```json
{
  "success": false,
  "total_platforms": 1,
  "successful_posts": 0,
  "failed_posts": 1,
  "results": [
    {
      "account_id": "sa_123456",
      "platform": "TWITTER",
      "success": false,
      "error": "Rate limit exceeded",
      "message": "Daily post limit reached for TWITTER"

  ]
}
```

#### 4. Invalid Platform
```json
{

```

## Rate Limits

- **Posts**: 30 per minute per account
- **Media Uploads**: 10 per minute per account
- **Max Media Size**: 60MB per file

The API tracks daily post counts per account and returns an error if limits are exceeded.

---

## Security Notes

1. **OAuth State**: The state parameter prevents CSRF attacks
2. **Access Tokens**: Should be encrypted in production
3. **User Validation**: All APIs verify account ownership before publishing
4. **Rate Limiting**: Prevents abuse and respects platform limits

## Troubleshooting

### OAuth Flow Not Working
- Verify `BLOTATO_CLIENT_ID` and `BLOTATO_CLIENT_SECRET` are correct
- Check `BASE_FRONTEND_URL` matches your callback URL in Blotato dashboard
- Ensure user is authenticated before initiating OAuth

### Post Publishing Failures
- Verify user has connected accounts via `/api/v1/social/accounts`
- Check account status is `is_active: true`
- Verify access tokens haven't expired (implement refresh token flow)
- Check post content length doesn't exceed platform limits

### Database Errors
- Run `alembic upgrade head` to ensure schema is up to date
- Verify PostgreSQL connection is working
- Check user_id foreign key relationships
